前言

不知道大家在学习ts的过程中，觉得难点在哪？我觉得一大难点里面的类型系统。 这里介绍下ts的内置类型。

Awaited<Type> Released: 4.5

获取 async 中的 await，或 Promise 中的 then 方法的返回类型。还能够获取嵌套的 Promise 返回值。

Example

1
2
3
4
5

type A = Awaited<Promise<string>>; // type A = string
    
type B = Awaited<Promise<Promise<number>>>; // type B = number
 
type C = Awaited<boolean | Promise<number>>; // type C = number | boolean

实现：

1
2
3
4
5
6
7
8

type Awaited<T> = T extends null | undefined  // 先判断是否null ｜ undefiend
  ? T 
  : T extends object & {
    then(onfulfilled: infer F): any; } // 再判断是对象且类promise对象
    ? F extends (value: infer V, ...args: any) => any // 获得then的value
      ? Awaited<V>  // 递归
      : never 
    : T

Partial<Type> Released: 2.1

把 Type 里的所有属性变成可选的属性的类型。

Example

17

interface Todo {
  title: string;
  description: string;
}
 
function updateTodo(todo: Todo, fieldsToUpdate: Partial<Todo>) {
  return { ...todo, ...fieldsToUpdate };
}
 
const todo1 = {
  title: "organize desk",
  description: "clear clutter",
};
 
const todo2 = updateTodo(todo1, {
  description: "throw out trash",
});

实现：

1
2
3

type Partial<T> = { 
  [P in keyof T]?: T[P] | undefined; // 通过?变成可选，如果要变成非可选，可以用-?
}

Required<Type> Released: 2.1

把 type 的所有属性变成必传属性的类型，Partial 的反义。

Example

  a?: number;
  b?: string;
}
 
const obj: Props = { a: 5 };

1

Readonly<Type> Released: 2.1

把 type 的所有属性变成只读属性的类型。

Example

  title: string;
}

1

Record<Keys, Type> Released: 2.1

以 Keys 为属性，Type 为值构建一个对象类型。通常用作类型映射。

Example

14

interface CatInfo {
  age: number;
  breed: string;
}
 
type CatName = "miffy" | "boris" | "mordred";
 
const cats: Record<CatName, CatInfo> = {
  miffy: { age: 10, breed: "Persian" },
  boris: { age: 5, breed: "Maine Coon" },
  mordred: { age: 16, breed: "British Shorthair" },
};
 
cats.boris; // const cats: Record<CatName, CatInfo>

实现：

1
2
3

type Record<K extends string | number | symbol, T> = { 
  [P in K]: T; 
}

Pick<Type, Keys> Released: 2.1

interface Todo {
  title: string;
  description: string;
  completed: boolean;
  createdAt: number;

type TodoPreview = Omit<Todo, "description">;

  title: "Clean room",
  completed: false,

type Omit<T, K extends string | number | symbol> = {

}

Exclude<UnionType, ExcludedMembers> Released: 2.8

从联合类型 UnionType 中移除指定类型 ExcludedMembers，以返回新的类型。 要注意和 Omit 区分，虽然作用差不多，但一个是针对对象，一个是针对联合类型。

Example

1
2
3
4
5

type T0 = Exclude<"a" | "b" | "c", "a">;
     
type T0 = "b" | "c"
type T1 = Exclude<"a" | "b" | "c", "a" | "b">; // type T1 = "c"
type T2 = Exclude<string | number | (() => void), Function>; // type T2 = string | number

实现：

1

type Exclude<T, U> = T extends U ? never : T // 联合类型会根据子类型自动迭代

Extract<Type, Union> Released: 2.8

从联合类型 Type 中取出 Union 中有的属性，可以简单理解为取交集。Exclude 的反义。

Example

1
2

type T0 = Extract<"a" | "b" | "c", "a" | "f">; // type T0 = "a"
type T1 = Extract<string | number | (() => void), Function>; // type T1 = () => void

实现：

1

type Extract<T, U> = T extends U ? T : never

Parameters<Type> Released: 3.1

从一个函数类型中，获取其参数元组类型。

Example

 1

19

declare function f1(arg: { a: number; b: string }): void;
 
type T0 = Parameters<() => string>; // type T0 = []
type T1 = Parameters<(s: string) => void>; // type T1 = [s: string]
type T2 = Parameters<<T>(arg: T) => T>; // type T2 = [arg: unknown]
type T3 = Parameters<typeof f1>;
// type T3 = [arg: {
//     a: number;
//     b: string;
// }]
type T4 = Parameters<any>; // type T4 = unknown[]
type T5 = Parameters<never>; // type T5 = never
type T6 = Parameters<string>; 
// Type 'string' does not satisfy the constraint '(...args: any) => any'.
// type T6 = never
type T7 = Parameters<Function>; 
// Type 'Function' does not satisfy the constraint '(...args: any) => any'.
// Type 'Function' provides no match for the signature '(...args: any): any'.
// type T7 = never

实现：

1
2
3

type Parameters<T extends (...args: any) => any> = T extends (...args: infer P) => any 
  ? P 
  : never // infer 推断

ConstructorParameters<Type> Released: 3.1

从一个构造函数类型中，获取其所有参数，以产生新的元祖类型。其中 Type 不是函数则返回 never。

Example

1
2
3
4
5
6
7
8
9

type T0 = ConstructorParameters<ErrorConstructor>; // type T0 = [message?: string]
type T1 = ConstructorParameters<FunctionConstructor>; // type T1 = string[]
type T2 = ConstructorParameters<RegExpConstructor>; // type T2 = [pattern: string | RegExp, flags?: string]
type T3 = ConstructorParameters<any>; // type T3 = unknown[]
 
type T4 = ConstructorParameters<Function>;
// Type 'Function' does not satisfy the constraint 'abstract new (...args: any) => any'.
// Type 'Function' provides no match for the signature 'new (...args: any): any'.
// type T4 = never

实现：

1
2
3

type ConstructorParameters<T extends abstract new (...args: any) => any> = T extends abstract new (...args: infer P) => any 
  ? P 
  : never

ThisParameterType<Type> Released: 3.3

返回函数类型 Type 中，this 的参数类型；如果没有 this ，则返回 unknown。

Example

1
2
3
4
5
6
7

function toHex(this: Number) {
  return this.toString(16);
}
 
function numberToString(n: ThisParameterType<typeof toHex>) {
  return toHex.apply(n);
}

实现：

1
2
3

type ThisParameterType<T> = T extends (this: infer U, ...args: never) => any 
  ? U 
  : unknown

OmitThisParameter<Type> Released: 3.3

从类型 Type 中，移除 this 参数。如果没有声明 this，则返回 Type；否则，返回一个新的不包含 this 入参的函数类型。

Example

1
2
3
4
5
6
7

function toHex(this: Number) {
  return this.toString(16);
}
 
const fiveToHex: OmitThisParameter<typeof toHex> = toHex.bind(5);
 
console.log(fiveToHex());

实现： this 做为一个虚拟参数，一般是作为调用上下文的推断。具体看这。

1
2
3
4
5

type OmitThisParameter<T> = unknown extends ThisParameterType<T> 
  ? T 
  : T extends (...args: infer A) => infer R 
    ? (...args: A) => R 
    : T

ThisType<Type> Released: 2.3

在 noImplicitThis 选项开启的情况下才能使用。这个操作不会返回类型，只用作上下文 this 的标记。

type ObjectDescriptor<D, M> = {
  data?: D;
  methods?: M & ThisType<D & M>; // Type of 'this' in methods is D & M
};
 
function makeObject<D, M>(desc: ObjectDescriptor<D, M>): D & M {
  let data: object = desc.data || {};
  let methods: object = desc.methods || {};
  return { ...data, ...methods } as D & M;
}
 
let obj = makeObject({
  data: { x: 0, y: 0 },
  methods: {
    moveBy(dx: number, dy: number) {
      this.x += dx; // Strongly typed this
      this.y += dy; // Strongly typed this
    },
  },
});
 
obj.x = 10;
obj.y = 20;
obj.moveBy(5, 5);

Intrinsic String Manipulation Types

字符串操作类型。

Uppercase<StringType>

把字符串都每个字符转换成大写。

Example

1
2
3
4
5

type Greeting = "Hello, world"
type ShoutyGreeting = Uppercase<Greeting> // type ShoutyGreeting = "HELLO, WORLD"
 
type ASCIICacheKey<Str extends string> = `ID-${Uppercase<Str>}`
type MainID = ASCIICacheKey<"my_app"> // type MainID = "ID-MY_APP"

Lowercase<StringType>

把字符串每个字符都转换成小写。

Example

1
2
3
4
5

type Greeting = "Hello, world"
type QuietGreeting = Lowercase<Greeting> // type QuietGreeting = "hello, world"
 
type ASCIICacheKey<Str extends string> = `id-${Lowercase<Str>}`
type MainID = ASCIICacheKey<"MY_APP"> // type MainID = "id-my_app"

Capitalize<StringType>

把字符串的第一个字符转换成大写。

Example

1
2

type LowercaseGreeting = "hello, world";
type Greeting = Capitalize<LowercaseGreeting>; // type Greeting = "Hello, world"

Uncapitalize<StringType>

把字符串的第一个字符转换成小写。

Example

1
2

type UppercaseGreeting = "HELLO WORLD";
type UncomfortableGreeting = Uncapitalize<UppercaseGreeting>; // type UncomfortableGreeting = "hELLO WORLD"
